Modals, Modals, and More Modals

Bootstrap modals are very simple to use, but how do you incorporate AngularJS functionality?
UI-bootstrap holds the answer.

Things to Note:
Order is EVERYTHING.
Well, maybe you can get away with unordered script tags and links, but reasons can convey otherwise.
Reason 1: It is said to be best practice to place script tags at the bottom of the body.
Why?
With multiple calls to other files, you don't want to keep your user waiting. Doing this allows the DOM to load syncrously, (top to bottom) with the CSS and HTML loading first, and functionality last. By the time all files are loaded, the user can interact with the web app as needed.
Reason 2: Long hours of debugging only to find the order of script tabs is necessary.
The link reference to the bootstrap.css is a necessity to make the Modal apparent. However, if placed after the static CSS reference for the html, the html styling will be cascaded over (overwritten, and forever corrupted). True story: After I contently styled my blog, it was almost unreadable when placed before the bootstrap.css link tag.
My modal template is on the HTML page. What if I wanted the template to be in my templates folder?
You simply have to copy everything within the script tags (not including the script tags), paste in another html file, and delete all existing evidence in the html file. Your html file should only require the button or tag that opens the modal when clicked. In the myController function, change the modal options inside ```modal.open({});```. The templateUrl should be set to the path of the new modal template.
Modals can incorporate services and factories as usual.
Modals can update the html environment based on the modal input.
Modals will be the best thing that has ever happened to your Angular application. (Says me)
